# Code Review

Review the specified code for issues and improvements.

## Arguments

- `$ARGUMENTS` - File path or description of code to review (optional, uses recent changes if not specified)

## Instructions

Review the code for:

1. **Bugs & Logic Errors** - Off-by-one errors, null handling, edge cases
2. **Security Issues** - Injection vulnerabilities, auth problems, data exposure
3. **Performance** - Inefficient algorithms, unnecessary operations, memory leaks
4. **Readability** - Naming, complexity, documentation needs
5. **Best Practices** - Language idioms, design patterns, SOLID principles

Format your review as:
- List issues by severity (Critical > High > Medium > Low)
- Include line numbers and specific suggestions
- Praise good patterns you notice
