# Generate Tests

Generate tests for the specified code.

## Arguments

- `$ARGUMENTS` - File path or function name to test

## Instructions

1. Read the specified file or function
2. Identify the testing framework used in the project (look for existing tests, package.json, pytest.ini, etc.)
3. Generate comprehensive tests including:
   - Happy path cases
   - Edge cases (empty inputs, null values, boundaries)
   - Error cases (invalid inputs, exceptions)
4. Follow the existing test patterns and naming conventions in the project
5. Place tests in the appropriate location based on project structure
