# Generate Documentation

Generate or update documentation for code.

## Arguments

- `$ARGUMENTS` - File, function, or module to document

## Instructions

1. Read the specified code thoroughly
2. Generate appropriate documentation:
   - **Functions/Methods**: Docstrings with params, returns, raises, examples
   - **Classes**: Class-level docs plus method docs
   - **Modules**: Module overview, public API summary
   - **APIs**: Endpoint docs with request/response examples
3. Match the documentation style already used in the project
4. Include practical examples where helpful
5. Document edge cases and important behaviors
