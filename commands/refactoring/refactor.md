# Refactor Code

Refactor code to improve quality without changing behavior.

## Arguments

- `$ARGUMENTS` - File path or description of what to refactor

## Instructions

1. Read and understand the current implementation
2. Identify refactoring opportunities:
   - Extract repeated code into functions
   - Simplify complex conditionals
   - Improve naming for clarity
   - Apply appropriate design patterns
   - Reduce coupling, increase cohesion
3. Make changes incrementally
4. Ensure behavior is preserved (run tests if available)
5. Explain what you changed and why
